关于柳比歇夫、尼古拉斯·卢曼和 GTD 之梦

ZERO-GTD 的目标是实现如下构想：

最终，它应当实现以下原则：

  1. 尽可能收集所有产生的想法
  2. 总是从已有的工作开展下一步工作
  3. 使用这个系统时应当像起床刷牙一样毫不费力
  4. 使大脑专注于工作，解除心智负担

以下是关于 Idea, Task, TimeChunk 的设计

public class Idea {
    
    private String name; // 名称
    private String summary; // 额外信息
    private int id; // 唯一 ID
    private int[] father_id; // 父想法的 ID
    private int locateTimeChunk; // 所在的时间区块
    private LocalDateTime initDateTime; // 初始化时间

    public Idea(String name, String summary); // 创建新的 Idea 对象
    public static void ideaView(); // 从数据库中查询并展示当前时间区块上的 Idea 实例
    public static void ideaView(TimeChunk certainTimeChunk); // 从数据库中筛选指定时间区块中保存的想法
    public static void ideaView(Object unknown); // 指定筛选
    public void ideaChainView(); // 展示想法链

    public void resetName(String name);
    public void resetSummary(String message);
    public void inheritIdea(Idea *thatidea); // 想法继承
       
    private void saveIdea(); // 将想法存储到数据库
}
public class Task {
    private String name; // 任务名
    private String summaryInMarkdown; // 任务细节，支持 Markdown 渲染
    private int pointToWhichIdea; // 所属的想法
    private boolean isFinished; // 任务完成状态
    private LocalDateTime initDateTime; // 初始化时间
    private LocalDateTime finishedDateTime; // 完成时间，默认为 2099年9月9日

    /**
     * 初始化任务并保存到数据库
     * @param name
     * @param summary
     * @param pointToWhichIdea
     */
    public Task(String name, String summary, Idea pointToWhichIdea) {};

    /**
     * 修改类方法
     * @param name
     */
    public void resetName(String name) {};
    public void resetSummaryInMarkdown(String summary) {};
    public void turnStatus(boolean yesOrNo) {};

    /**
     * 查询类方法
     */
    public static void getOneDayTaskList() {};
    public static void showCertainTaskList() {};
    // 删除类方法
    public static void deleteTask() {};

    private void saveTask() {}; // 将任务保存到数据库
}

TimeChunk 类中有些细节还没有考虑清楚，比如 TimeChunk 作为整体和单个时间条目的关系，应该如何储存和修改, 因此 TimeChunk 类的设计留待更新
